# 项目代码浏览 UI 增强设计（2026-03-19）

## 背景与目标
- 用户希望在项目代码浏览页为“打开文件”增加就地搜索 UI，不切换搜索模式也能快速定位文件。
- 左侧文件面板与右侧代码预览面板的外线需要更明显，提升两个主工作区的边界感。
- 右侧代码显示需要增强可读性，包括字体、行高、命中提示与焦点行表现。

## 范围
- `frontend/src/pages/ProjectCodeBrowser.tsx`
- `frontend/src/pages/AgentAudit/components/FindingCodeWindow.tsx`
- 对应页面测试与展示预设

## 方案选择
- 采用 A 方案：在左侧“打开文件”面板顶部增加快速搜索输入框，保留现有“文件 / 搜索”模式切换。
- `文件` 模式负责“快速打开文件”，优先按文件名与路径即时过滤树。
- `搜索` 模式继续承担文件内容搜索、包含/排除过滤与结果跳转。

## 布局与交互设计
1. **左侧文件面板**
   - 在“打开文件”标题下加入一个快速搜索输入框，placeholder 为“搜索文件名 / 路径”。
   - 输入后即时过滤文件树，匹配规则为：`trim + 不区分大小写`，同时匹配文件名与完整路径，不做 debounce。
   - 过滤结果保留目录层级，只隐藏无命中的文件与空目录；任一命中文件的祖先目录在过滤态下自动展开，避免结果被折叠遮住。
   - 当过滤有结果时继续支持点击打开文件；无结果时显示轻量空态提示。
2. **模式结构**
   - 保留现有左侧模式轨：`文件` 与 `搜索`。
   - `文件` 模式顶部展示快速打开搜索框；`搜索` 模式维持全文搜索输入、包含目录与排除目录输入。
3. **双面板外线**
   - 左右主面板保持现有布局比例不变。
   - 两个主面板容器从当前偏弱边框提升为“更亮的主边线 + 轻微内描边/阴影”的组合，增强区块感。
   - 验收参考：边线视觉强度需明显高于当前 `border-white/10`，建议提升到接近 `border-white/14` 级别并增加一层轻微描边或阴影。

## 视觉设计
1. **左侧文件面板**
   - 搜索框沿用当前深色终端风格，但提高边框对比度和 focus 状态亮度。
   - 选中文件项边界与背景高亮略增强，与新搜索框风格保持一致。
2. **右侧代码预览**
   - `project-browser` 预设的代码字体与行高提升，减少阅读压迫感。
   - 命中关键词增加更清晰的高亮底色，焦点行保留更强的背景层次与左侧强调条。
   - 行号列与代码正文的明暗分层进一步拉开，避免整块代码发灰。
   - 本轮不引入完整语法高亮引擎，增强范围限定为：字号、行高、对比度、行号区分层、命中关键词高亮与焦点行强调。
3. **空态与反馈**
   - 文件模式下如果搜索没有结果，显示“未找到匹配文件”类空态。
   - 保持现有加载/失败态文案，不改变业务逻辑。

## 实施步骤
1. 在 `ProjectCodeBrowserContent` / `Workspace` 中新增文件模式搜索状态，并将其传入左侧文件面板。
2. 扩展文件树过滤逻辑，为文件模式构建“按路径/文件名筛选”的派生树；该派生树仅服务文件模式显示，不能修改搜索模式的数据集与结果状态。
3. 更新左侧文件面板 UI，加入快速搜索输入框与无结果空态。
4. 调整左右主面板外框样式，使边界更清晰但不过度发光。
5. 更新 `FindingCodeWindow` 的 `project-browser` 预设，增强代码显示与高亮表现。

## 风险与缓解
- **风险**：文件树过滤破坏目录上下文。  
  **缓解**：过滤时保留祖先目录，仅裁剪无命中叶子和空目录。
- **风险**：边线增强过重，破坏现有深色终端风格。  
  **缓解**：以对比度提升为主，只增加轻微描边与阴影，不引入夸张霓虹效果。
- **风险**：代码字体放大导致长行横向滚动增加。  
  **缓解**：仅小幅提升字号与行高，优先改善对比度和高亮层次。

## 验收标准
- 文件模式左侧顶部可直接搜索文件名/路径，并即时过滤当前文件树。
- 左右两个主面板外线比当前更明显，仍保持统一视觉风格。
- 右侧代码区的字体、行高、命中关键词和焦点行表现均较当前更清晰。
- 页面测试和类型检查通过。
